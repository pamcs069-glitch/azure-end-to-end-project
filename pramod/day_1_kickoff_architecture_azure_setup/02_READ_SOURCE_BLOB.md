## Key Terms — Read This Before Running

### `wasbs://` vs `abfss://`

| | `abfss://` | `wasbs://` |
|---|---|---|
| Full name | Azure Data Lake Storage Gen2 | Windows Azure Storage Blob Service |
| Auth method | OAuth (Service Principal login) | SAS token or Access Key |
| Requires Hierarchical Namespace | Yes | No |
| When to use | Your own ADLS Gen2 containers | External / classic blob storage |

**Rule:** If the storage belongs to someone else and they gave you a SAS token → use `wasbs://`.

---

### What is a SAS Token?

SAS = **Shared Access Signature**. It is a long string of URL parameters that proves you are allowed to access specific resources in the storage account.

A SAS token looks like this:
```
se=2027-07-30&sp=rl&spr=https&sv=2026-04-06&sr=c&sig=EeNl1mQL9JN...
```

Each part means:

| Parameter | Meaning |
|---|---|
| `se=2027-07-30` | **Expiry** — token stops working after this date |
| `sp=rl` | **Permissions** — `r` = read, `l` = list. You can read files and list folders. Cannot write or delete. |
| `spr=https` | **Protocol** — HTTPS only, not plain HTTP |
| `sv=2026-04-06` | **API version** of the Azure storage service |
| `sr=c` | **Scope** — `c` means this token covers the entire container |
| `sig=...` | **Signature** — cryptographic proof the token was generated by the storage account owner |

---

### `dbutils.fs.ls()`

- `dbutils` = Databricks Utilities — a helper library built into every Databricks notebook
- `dbutils.fs` = the file system section of those utilities
- `dbutils.fs.ls(path)` = lists files and folders at a given path, same idea as `ls` in Linux

It returns a list of `FileInfo` objects. Each object has:
- `.name` — the file or folder name only, e.g. `"2026/"`
- `.path` — the full path, e.g. `"wasbs://source@dataenggdailystorage.blob.core.windows.net/2026/"`
- `.size` — size in bytes. Folders always show `0`.

---

### `spark.conf.set()`

- `spark` = the active Spark session — your entry point to all Spark operations
- `spark.conf.set(key, value)` = sets a configuration value for this session only

It is **not permanent** — the config is cleared when the cluster restarts. You must re-run Cell 1 each session.

The key pattern `fs.azure.sas.{container}.{account}.blob.core.windows.net` tells Spark's Azure connector: when you see a `wasbs://source@dataenggdailystorage...` path, use the SAS token to authenticate.

## Cell 1 — Load credentials from Key Vault + configure Spark

**What it does:**
1. Reads 3 secrets from Key Vault — nothing is hardcoded in the notebook
2. Calls `spark.conf.set()` to register the SAS token with Spark for this session

**Line by line:**
- `SCOPE` — the name of the Databricks secret scope linked to your Key Vault. Acts as a bridge: `"kv-ev-scope"` → your Key Vault → the actual secret value.
- `dbutils.secrets.get(scope, key)` — reads a secret from Key Vault. The value never appears in logs or print output — Databricks always shows `[REDACTED]`. The `key=` must exactly match the secret name in Key Vault.
- `STORAGE_ACCOUNT` — resolves to `"dataenggdailystorage"` (the external account, not yours)
- `CONTAINER` — resolves to `"source"`, the container name inside that storage account
- `SAS_TOKEN` — the full token string starting with `se=...`
- `spark.conf.set(...)` — the key `fs.azure.sas.{CONTAINER}.{STORAGE_ACCOUNT}.blob.core.windows.net` tells Spark which SAS token to use when it sees a `wasbs://` path for this account. Session-scoped only — cleared on restart.

```python
SCOPE = "kv-ev-scope"

STORAGE_ACCOUNT = dbutils.secrets.get(scope=SCOPE, key="source-storage-account")
CONTAINER       = dbutils.secrets.get(scope=SCOPE, key="source-container")
SAS_TOKEN       = dbutils.secrets.get(scope=SCOPE, key="source-sas-token")

print(f"Storage account : {STORAGE_ACCOUNT}")
print(f"Container       : {CONTAINER}")
print(f"SAS token       : [REDACTED]")
print("Credentials loaded from Key Vault — OK")

spark.conf.set(
    f"fs.azure.sas.{CONTAINER}.{STORAGE_ACCOUNT}.blob.core.windows.net",
    SAS_TOKEN
)
print("Spark SAS config set — OK")
```

**Expected output:**
```
Storage account : dataenggdailystorage
Container       : source
SAS token       : [REDACTED]
Credentials loaded from Key Vault — OK
Spark SAS config set — OK
```

**Errors:**

| Error | Cause | Fix |
|---|---|---|
| `Secret does not exist` | Secret name wrong or not created yet | Add secrets to Key Vault — see Prerequisites above |
| `Secret scope not found` | `kv-ev-scope` not created | Day 1 Part 6.5 — create the secret scope first |

## Cell 2 — List top-level folders in the source container

**What it does:** Lists everything at the root of the `source` container — like running `ls /` on the container.

**Line by line:**
- `base_path` — the full root URL of the container. `wasbs://` is the protocol, `{CONTAINER}@{STORAGE_ACCOUNT}.blob.core.windows.net` is the domain, the trailing `/` means start from the root.
- `dbutils.fs.ls(base_path)` — lists all files and folders at that path. Returns a list of `FileInfo` objects.
- `item.size == 0` — folders always have size 0. Files have a size greater than 0.
- `item.name` — name of the folder or file, e.g. `"realtime/"`.

```python
base_path = f"wasbs://{CONTAINER}@{STORAGE_ACCOUNT}.blob.core.windows.net/"

items = dbutils.fs.ls(base_path)

for item in items:
    item_type = "DIR " if item.size == 0 else "FILE"
    print(f"  [{item_type}] {item.name:<50} {item.size:>10} bytes")

print(f"\n  Total: {len(items)} items found at root level")
```

**Expected output:**
```
  [DIR ] realtime/                                              0 bytes
  [DIR ] historical/                                            0 bytes

  Total: 2 items found at root level
```

**Errors:**

| Error | Cause | Fix |
|---|---|---|
| `403 Forbidden` | SAS token wrong, expired, or missing `l` (list) permission | Check `sp=rl` is in the token. Re-add correct token to Key Vault. |
| `AuthenticationFailed` | Spark SAS config not set | Re-run Cell 1 first |

## Cell 3 — Drill into the charging_sessions folder

**What it does:** Goes one level deeper into `realtime/charging_sessions/` to see exactly how files are partitioned. You need this information to write the correct glob pattern in Cell 5.

**Line by line:**
- `sessions_root` — the path to the `charging_sessions/` subfolder inside the container
- First `dbutils.fs.ls(sessions_root)` — lists Level 1 contents, e.g. year folders like `2026/`
- `item.path` — the full path of the item, needed to `ls` into it (`.name` alone is not enough)
- Second `dbutils.fs.ls(item.path)` — lists Level 2 contents inside each year folder, e.g. `06/` for June
- The inner `try/except pass` — if an item is a file rather than a folder, `ls()` will error. We silently skip it.

```python
sessions_root = (
    f"wasbs://{CONTAINER}@{STORAGE_ACCOUNT}.blob.core.windows.net"
    f"/realtime/charging_sessions/"
)

items = dbutils.fs.ls(sessions_root)
print("Level 1 — contents of charging_sessions/:")
for item in items:
    print(f"  {item.name}")

print("\nLevel 2 — drilling one folder deeper:")
for item in items:
    try:
        sub_items = dbutils.fs.ls(item.path)
        for s in sub_items:
            print(f"  {item.name}{s.name}")
    except:
        pass
```

**Expected output:**
```
Level 1 — contents of charging_sessions/:
  2026/
Level 2 — drilling one folder deeper:
  2026/06/
```

This tells you files sit at: `charging_sessions/2026/06/DD/HH/file.csv` → **4 levels deep**.

## Cell 4 — Read one specific CSV file

**What it does:** Reads a single known CSV file into a Spark DataFrame and shows its schema and first 10 rows.

**Line by line:**
- `csv_path` — the full `wasbs://` URL to one specific file. Adjust the date/hour based on what Cell 3 showed.
- `spark.read` — entry point to load data into Spark. You chain `.option()` calls and a format method onto it.
- `.option("header", "true")` — the first row of the CSV is column names, not a data row.
- `.option("inferSchema", "true")` — Spark scans the file to auto-detect column types (integer, double, string, date, etc.). Without this, every column is treated as plain text.
- `.csv(csv_path)` — reads the file as a CSV and returns a DataFrame. Nothing is actually read yet — Spark is lazy.
- `df` — a Spark DataFrame: a distributed table held across cluster workers.
- `df.count()` — triggers the actual file read. Counts all rows.
- `df.printSchema()` — prints column names and their detected types in a tree view.
- `display(df.limit(10))` — Databricks helper that shows an interactive table. `.limit(10)` fetches only 10 rows instead of scanning the whole file.

```python
csv_path = (
    f"wasbs://{CONTAINER}@{STORAGE_ACCOUNT}.blob.core.windows.net"
    f"/realtime/charging_sessions/2026/06/01/06/sessions_20260601_0600.csv"
)

df = (
    spark.read
    .option("header", "true")
    .option("inferSchema", "true")
    .csv(csv_path)
)

print(f"Row count    : {df.count():,}")
print(f"Column count : {len(df.columns)}")
print(f"Columns      : {df.columns}\n")
df.printSchema()
display(df.limit(10))
```

**Expected output:**
```
Row count    : 1,250
Column count : 8
Columns      : ['session_id', 'start_time', 'end_time', 'energy_kwh', ...]

root
 |-- session_id: string (nullable = true)
 |-- energy_kwh: double (nullable = true)
 ...
```

**Errors:**

| Error | Cause | Fix |
|---|---|---|
| `FileNotFoundException` | Date/hour in path is wrong | Check Cell 3 output for correct folder names |
| `403 Forbidden` | SAS token missing `r` (read) permission | Confirm `sp=rl` is in the token |
| `abfss error` | Used `abfss://` instead of `wasbs://` | Change the protocol prefix to `wasbs://` |

## Cell 5 — Read ALL CSV files at once (glob pattern)

**What it does:** Reads every CSV file across all year/month/day/hour subfolders in a single Spark job.

**What is a glob pattern?**
A `*` in a path means "match anything at this one level":
- `charging_sessions/*` → matches any year folder (2025, 2026, ...)
- `charging_sessions/*/*` → matches any year/month combination
- `charging_sessions/*/*/*/*/*.csv` → matches any CSV file exactly 4 folder levels deep

**Line by line:**
- `base` — the root URL of the storage account and container, without any path yet
- `glob_path` — the full path with wildcards. The four `/*` represent: year / month / day / hour. The final `/*.csv` matches any CSV file at that depth.
- `spark.read.csv(glob_path)` — Spark finds all matching files in parallel, reads them, and automatically unions them into one single DataFrame.
- `df_all` — one combined DataFrame containing rows from every matched file.
- `df_all.count()` — triggers the actual read and union across all files.

```python
base      = f"wasbs://{CONTAINER}@{STORAGE_ACCOUNT}.blob.core.windows.net"
glob_path = f"{base}/realtime/charging_sessions/*/*/*/*/*.csv"

df_all = (
    spark.read
    .option("header", "true")
    .option("inferSchema", "true")
    .csv(glob_path)
)

total_rows = df_all.count()
print(f"Total rows across all files : {total_rows:,}")
print(f"Column count                : {len(df_all.columns)}")
display(df_all.limit(10))
```

**Expected output:**
```
Total rows across all files : 45,230
Column count                : 8
```

**If you get a folder-depth error:** count the levels between `charging_sessions/` and the `.csv` files from Cell 3 output, then adjust:
- 3 levels deep → `/*/*/*/*.csv`
- 4 levels deep → `/*/*/*/*/*.csv` (current setting)

## Cell 6 — Data quality check

**What it does:** Scans every column for null and empty values. Flags any column where more than 10% of rows are missing.

**Line by line:**
- `import pyspark.sql.functions as F` — imports Spark's built-in column functions. `F` is the standard alias everyone uses.
- `F.col(col_name)` — a reference to a column by name. Does not read data yet — Spark is lazy.
- `.isNull()` — returns True for a row if the value is completely missing (NULL).
- `.cast("string")` — temporarily converts the column value to text so we can check for empty string `""`, even on numeric columns.
- `|` — the OR operator. The filter flags a row if the value is null OR is an empty string.
- `.filter(condition)` — keeps only rows where the condition is True. Same as SQL `WHERE`.
- `.count()` — counts the filtered rows. This is what actually triggers Spark to compute.
- `pct > 10` — any column with more than 10% nulls/empty values gets a flag for investigation.

```python
import pyspark.sql.functions as F

total = df_all.count()
print(f"Total rows : {total:,}\n")

for col_name in df_all.columns:
    null_count = df_all.filter(
        F.col(col_name).isNull() | (F.col(col_name).cast("string") == "")
    ).count()

    pct  = (null_count / total * 100) if total > 0 else 0
    flag = "<-- check this" if pct > 10 else ""
    print(f"  {col_name:<35} {null_count:>8,}  {pct:>5.1f}%  {flag}")
```

**Expected output:**
```
  Column                              Nulls       %
  session_id                              0    0.0%
  energy_kwh                          2,340    5.2%
  station_id                              0    0.0%
  customer_id                         4,800   10.6%  <-- check this
```

Columns flagged `<-- check this` will be cleaned in the Silver layer on Day 7.

## Quick Reference — Protocol Cheat Sheet

| Situation | Protocol | Auth |
|---|---|---|
| Your own ADLS Gen2 (Bronze/Silver/Gold) | `abfss://` | SP OAuth via `spark.conf.set` |
| External blob with SAS token | `wasbs://` | SAS token via `spark.conf.set` |
| External blob with Access Key | `wasbs://` or `abfss://` | Account key via `spark.conf.set` |
| Public blob (no auth needed) | `wasbs://` | None |
