## Key Terms ‚Äî Read This Before Running

### `wasbs://` vs `abfss://`

| | `abfss://` | `wasbs://` |
|---|---|---|
| Full name | Azure Data Lake Storage Gen2 | Windows Azure Storage Blob Service |
| Auth method | OAuth (Service Principal login) | SAS token or Access Key |
| Requires Hierarchical Namespace | Yes | No |
| When to use | Your own ADLS Gen2 containers | External / classic blob storage |

**Rule:** If the storage belongs to someone else and they gave you a SAS token ‚Üí use `wasbs://`.

---

### What is a SAS Token?

SAS = **Shared Access Signature**. It is a long string of URL parameters that proves you are allowed to access specific resources in the storage account.

A SAS token looks like this:
```
se=2027-07-30&sp=rl&spr=https&sv=2026-04-06&sr=c&sig=EeNl1mQL9JN...
```

Each part means:

| Parameter | Meaning |
|---|---|
| `se=2027-07-30` | **Expiry** ‚Äî token stops working after this date |
| `sp=rl` | **Permissions** ‚Äî `r` = read, `l` = list. You can read files and list folders. Cannot write or delete. |
| `spr=https` | **Protocol** ‚Äî HTTPS only (not plain HTTP) |
| `sv=2026-04-06` | **API version** of the Azure storage service |
| `sr=c` | **Scope** ‚Äî `c` means this token is for the entire **container** |
| `sig=...` | **Signature** ‚Äî cryptographic proof the token was generated by the storage account owner |

---

### `dbutils.fs.ls()`

`dbutils` = **Databricks Utilities** ‚Äî a helper library built into every Databricks notebook.
`dbutils.fs` = the **file system** section of those utilities.
`dbutils.fs.ls(path)` = **list** files and folders at `path`, like `ls` in Linux.

Returns a list of `FileInfo` objects. Each has:
- `.name` ‚Äî just the file/folder name (e.g. `"2026/"`)
- `.path` ‚Äî the full path (e.g. `"wasbs://source@..."`)
- `.size` ‚Äî size in bytes. `0` usually means it is a folder.

---

### `spark.conf.set()`

`spark` = the active Spark session ‚Äî your entry point to all Spark operations.
`spark.conf.set(key, value)` = sets a configuration value for this Spark session only.

It is **not permanent** ‚Äî cleared when the cluster restarts. You must re-run Cell 1 each session.

The key `fs.azure.sas.{container}.{account}.blob.core.windows.net` tells Spark's Azure connector:
> "When you see a `wasbs://source@dataenggdailystorage...` path, use this SAS token to authenticate."

## Cell 1 ‚Äî Load credentials from Key Vault + configure Spark

**What it does:**
1. Reads 3 secrets from Key Vault using `dbutils.secrets.get()`
2. Calls `spark.conf.set()` to tell Spark which SAS token to use for this storage account

```python
SCOPE = "kv-ev-scope"
# SCOPE = the name of the Databricks secret scope linked to your Key Vault.
# Think of it as: "kv-ev-scope" ‚Üí bridge ‚Üí Key Vault ‚Üí actual secret value

STORAGE_ACCOUNT = dbutils.secrets.get(scope=SCOPE, key="source-storage-account")
# dbutils.secrets.get() reads a secret WITHOUT ever showing the value.
# Even if you print it, Databricks shows [REDACTED] ‚Äî value never appears in logs.
# key= must match exactly the secret name you created in Key Vault.

CONTAINER = dbutils.secrets.get(scope=SCOPE, key="source-container")
# "source" ‚Äî the container name inside dataenggdailystorage

SAS_TOKEN = dbutils.secrets.get(scope=SCOPE, key="source-sas-token")
# The full SAS token string: se=2027-07-30&sp=rl&spr=https&...

spark.conf.set(
    f"fs.azure.sas.{CONTAINER}.{STORAGE_ACCOUNT}.blob.core.windows.net",
    SAS_TOKEN
)
# Key breakdown:
#   fs.azure.sas          ‚Üí "use SAS token auth for this Azure blob path"
#   .{CONTAINER}          ‚Üí which container (source)
#   .{STORAGE_ACCOUNT}    ‚Üí which storage account (dataenggdailystorage)
#   .blob.core.windows.net ‚Üí Azure blob endpoint domain
#
# This config is session-scoped only ‚Äî cleared on cluster restart.
```

**Expected output:**
```
Storage account : dataenggdailystorage
Container       : source
SAS token       : [REDACTED]
Credentials loaded from Key Vault ‚Äî OK
Spark SAS config set ‚Äî OK
```

**Errors:**

| Error | Cause | Fix |
|---|---|---|
| `Secret does not exist` | Secret name wrong or not created yet | Add secrets to Key Vault ‚Äî see Prerequisites above |
| `Secret scope not found` | `kv-ev-scope` not created | Day 1 Part 6.5 ‚Äî create the secret scope first |

## Cell 2 ‚Äî List top-level folders in the source container

**What it does:** Lists everything at the root of the `source` container ‚Äî like running `ls /` on the container.

```python
base_path = f"wasbs://{CONTAINER}@{STORAGE_ACCOUNT}.blob.core.windows.net/"
# wasbs://    = protocol (use this with SAS tokens, NOT abfss://)
# {CONTAINER}@{STORAGE_ACCOUNT}.blob.core.windows.net = the full domain of the container
# Trailing /  = start from the root

items = dbutils.fs.ls(base_path)
# Lists all files and folders at this path.
# Returns list of FileInfo: each has .name, .path, .size

for item in items:
    item_type = "DIR " if item.size == 0 else "FILE"
    # size == 0 typically means it's a folder ‚Äî folders have no byte size
    print(f"  [{item_type}] {item.name:<50} {item.size:>10} bytes")
```

**Expected output:**
```
  [DIR ] realtime/                                              0 bytes
  [DIR ] historical/                                            0 bytes
  Total: 2 items found at root level
```

**Errors:**

| Error | Cause | Fix |
|---|---|---|
| `403 Forbidden` | SAS token wrong, expired, or `sp` missing `l` (list) | Check `sp=rl` is in the token. Re-add correct token to Key Vault. |
| `AuthenticationFailed` | Spark config not set | Re-run Cell 1 first |

## Cell 5 ‚Äî Read ALL CSV files at once (glob pattern)

**What it does:** Reads every CSV across all year/month/day/hour subfolders in one Spark job.

**What is a glob pattern?**
A `*` matches anything at that folder level:
- `/*` = any one folder name
- `/*/*` = any two levels of folder names
- `/*/*/*/*/*.csv` = any CSV file exactly 4 levels deep

```python
base = f"wasbs://{CONTAINER}@{STORAGE_ACCOUNT}.blob.core.windows.net"

glob_path = f"{base}/realtime/charging_sessions/*/*/*/*/*.csv"
# Pattern breakdown:
#   charging_sessions/  ‚Üí fixed starting folder
#   *                   ‚Üí any year  (e.g. 2026)
#   /*                  ‚Üí any month (e.g. 06)
#   /*                  ‚Üí any day   (e.g. 01)
#   /*                  ‚Üí any hour  (e.g. 06)
#   /*.csv              ‚Üí any CSV file at that depth

df_all = (
    spark.read
    .option("header", "true")
    .option("inferSchema", "true")
    .csv(glob_path)
    # Spark finds ALL matching files in parallel, reads them, and unions
    # them into one single DataFrame automatically
)

df_all.count()  # triggers read + union across all files
```

**Expected output:**
```
Total rows across all files : 45,230
Column count                : 8
```

**If you get an error about folder depth:** count the levels between `charging_sessions/` and the actual `.csv` files in Cell 3 output. Adjust the number of `/*` accordingly:
- 3 levels ‚Üí `/*/*/*/*.csv`
- 4 levels ‚Üí `/*/*/*/*/*.csv`

## Cell 6 ‚Äî Data quality check

**What it does:** Scans every column for null and empty values. Flags columns where more than 10% of rows are missing.

```python
import pyspark.sql.functions as F
# F is a short alias for pyspark.sql.functions ‚Äî the standard convention
# It gives you F.col(), F.isNull(), F.count(), etc.

for col_name in df_all.columns:
    null_count = df_all.filter(
        F.col(col_name).isNull()                          # value is NULL
        | (F.col(col_name).cast("string") == "")         # OR value is empty string ""
    ).count()
    # F.col(col_name)   ‚Üí reference to the column (not the data yet ‚Äî Spark is lazy)
    # .isNull()         ‚Üí True if the value is missing
    # .cast("string")   ‚Üí temporarily treat the value as text so we can compare to ""
    # |                 ‚Üí OR operator ‚Äî flag if EITHER condition is true
    # .filter()         ‚Üí keep only rows where condition is True (like SQL WHERE)
    # .count()          ‚Üí now count those rows ‚Äî triggers actual Spark computation
```

**Expected output:**
```
  Column                              Nulls       %
  session_id                              0    0.0%
  energy_kwh                          2,340    5.2%
  station_id                              0    0.0%
  customer_id                         4,800   10.6%  <-- check this
```

Columns flagged `<-- check this` will be handled in the Silver layer on Day 7.

## Quick Reference ‚Äî Protocol Cheat Sheet

| Situation | Protocol | Auth |
|---|---|---|
| Your own ADLS Gen2 (Bronze/Silver/Gold) | `abfss://` | SP OAuth via `spark.conf.set` |
| External blob with SAS token | `wasbs://` | SAS via `spark.conf.set` |
| External blob with Access Key | `wasbs://` or `abfss://` | Account key via `spark.conf.set` |
| Public blob (no auth) | `wasbs://` | None needed |
